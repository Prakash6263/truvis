System Overview
- Primary Function: Customer account access, balance view, fund transfers, card management.
- Environment: Public internet access via a responsive web application.
- Users: Retail customers and internal support agents.

Technology Stack
- Frontend: React 18, Tailwind CSS, served via CDN.
- Backend: Node.js 20 (Express), REST APIs.
- Database: PostgreSQL 14 for transactional data; Redis for sessions.
- Infra: Kubernetes on managed cloud, NGINX ingress, Cloud WAF enabled.
- Third-Parties: Payment Processor (PCI-DSS scope), Email/SMS provider.

Data Classification & Stores
- PII: Full name, email, phone (moderate sensitivity).
- Sensitive: Account numbers, transaction history (high sensitivity).
- Secrets: API keys managed in cloud secret manager.
- Logs: Centralized in ELK; some fields partially masked.

Authentication & Authorization
- Auth: Username/password; MFA enforced for admins. Customer MFA rollout planned within 6 months.
- Sessions: HTTP-only secure cookies; 24h expiry.
- RBAC: Customer, Support Agent, Admin roles.

Security Controls
- Transport: TLS 1.2+ enforced; HSTS enabled.
- At-Rest: Volume-level encryption via cloud provider. Field-level encryption for PII is not yet implemented.
- WAF: Managed rules active; custom rules for credential stuffing not tuned yet.
- Scanning: Weekly SAST; monthly DAST. Container images scanned on push.
- Backups: Nightly; restore tested quarterly.

Third-Party & Compliance
- Payment Processor: PCI-DSS SAQ A-EP evidence pending. Current 3rd-party audit report status is unclear.
- Email/SMS: DPA signed; geo EU; verification pending.
- Privacy: DSAR handled by separate service; data retention policy 7 years.

Known Gaps & Assumptions (Intentionally Ambiguous)
- Field-level encryption for PII is planned but not implemented (Q4 target).
- MFA for all customers in staged rollout; exact completion date TBD.
- Incident runbooks draft exists; tabletop exercise not yet performed.
- Cloud WAF custom rules for credential-stuffing are not tuned.
- Vendor audit status for the payment processor pending confirmation.
